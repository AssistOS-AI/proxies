Specs

Specifications

Capability-focused design specifications for Soul Gateway. Each spec is the authoritative source for its domain. Start with the README for the current middleware-first runtime model and the historical importer path.

Design Specifications

DS001

Request Pipeline

Request lifecycle for the three accepted formats (Chat Completions, Anthropic Messages, OpenAI Responses). Bearer auth, identity headers, request IDs, validation, streaming vs non-streaming, and the full pipeline phases.

DS002

Provider Authentication

Auth strategies (api_key / oauth / subscription), the five managed OAuth flows, multi-account pooling, token refresh with inline safety net, auto-provisioning, format converters, the provider template catalog, provider transport ownership invariant, and search providers as normal models.

DS003

Middleware, Backends & Extensions

Kernel middleware contract, gateway/provider/backend scopes, runtime context, the unified backend catalog, and extension discovery.

DS004

Model Routing

Model-name normalization, direct vs cascade routing, cooldowns, concurrency control, pricing, and the dashboard Tiers surface over cascade models.

DS005

Streaming

SSE pass-through to clients with internal stream tap for logging, non-streaming buffering, tool-call streaming, usage and finish-reason normalization, and real-time log broadcasting via WebSocket and SSE.

DS006

Database Schema

Table families (provider configuration, model registry, API keys, middleware, observability), encryption note (SQLite BLOB values), and single-table audit log retention.

DS007

Rate Limiting, Budgets & API Keys

Per-key RPM/TPM limits, daily and monthly budgets, spend caching, API-key lifecycle, and middleware-level override settings.

DS008

Content Filtering

Pre-dispatch blacklist rules, buffered response filtering, and the current limits of gateway-level filtering in client-streaming mode.

DS009

Error Handling

Error classification taxonomy, the shared error envelope, HTTP-level retry with exponential backoff and jitter, model-level cascade, account rotation on quota exhaustion, and retry logging.

DS010

Agent Loop Detector

Response-fingerprint and token-growth loop detection, session-scoped in-memory state, and the three supported response modes.

DS012

Management API & Dashboard

Dashboard authentication, provider/model/tier management, unified middleware binding APIs, connectivity tests, and the provider composer.

DS013

Configuration & Deployment

Environment variables, application defaults, self-initialization sequence on startup, achillesAgentLib discovery vs direct-provider modes, production deployment, health check, graceful shutdown, and encryption key management.

DS014

Built-in Middlewares

Per-middleware capability documentation for the 12 gateway middlewares that ship with the runtime: cache, rate limiter, budget enforcer, content blocker, loop detector, context compressor, prompt injector, session context, token tracker, request logger, response filter, output compressor.

DS015

Observability

Audit logging with full request context, real-time log streaming over WebSocket and SSE, metrics dashboards (cost, usage, error, activity, token, system), session and agent tracking, and bulk data export.
