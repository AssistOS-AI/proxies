# Remove Dashboard Logout + Connection-Status Indicator Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Remove the Logout button and the Live/Offline connection-status cluster from the Soul Gateway management dashboard, and delete the now-orphaned `/management/auth/*` 410 compatibility stubs, since the dashboard runs inside Explorer under Ploinky authentication.

**Architecture:** Two independent deletions. (1) Frontend: strip the `navbar-end` cluster (status badge + WS/SSE indicator + Logout button) from `index.html` and the dead `logout()` Alpine method from `app.mjs`. (2) Backend: delete the `/management/auth/login|logout|session` 410 stub routes (`auth-route.mjs`, their registrations in `build-routes.mjs`) plus the unit/integration tests that pin the 410 behavior, then record the removal in DS016. No new behavior is introduced; the dashboard already redirects to the Ploinky router's `/auth/login` and `/auth/logout`, which are untouched.

**Tech Stack:** Node.js ES modules, `node:test` (`--experimental-test-module-mocks`), Alpine.js + daisyUI/Tailwind dashboard (no JS unit harness for the dashboard template).

## Global Constraints

- ES modules with `import` / `export`; four-space indentation for JS; two-space for JSON/YAML. (Copied verbatim from `soul-gateway/CLAUDE.md`.)
- Keep all edits scoped to `proxies/soul-gateway/`. Do not touch sibling repos, `node_modules/`, or unrelated untracked files.
- Commits are human-authored. Never add `Co-Authored-By`, `Generated by`, tool-signature footers, or any AI/coding-agent attribution to commit messages or any project record.
- **Do NOT remove or modify** the Ploinky auth integration: the 401/403 → `/auth/login` redirect in `app.mjs` (lines ~149–163) and the Ploinky router's `/auth/login` / `/auth/logout` endpoints. This change only stops the dashboard from *linking* to logout.
- **Do NOT remove** the `wsConnected` / `streamMode` Alpine state or the WS/SSE connection lifecycle in `app.mjs` (lines ~584–585, 650–681). They drive live log streaming; only their visual binding is removed.
- This is a deletion-only change. There is no dashboard JS test harness, so Task 1 is verified by grep assertions + the existing test suite staying green + a visual check. Task 2 has real tests and uses a red→remove cycle.
