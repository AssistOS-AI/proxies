Soul Gateway Configuration
Reference

Environment Variables

Every environment variable recognized by Soul Gateway, sourced from the Ploinky manifest (manifest.json) and the application config module (src/config.mjs).

Manifest Environment Variables

These variables are declared in manifest.json under profiles.default.env. Ploinky injects them into the container at startup. All are optional.

VariableDefaultDescription
PORT 8042 HTTP server listen port.
UPSTREAM_URL https://proxy.axiologic.dev CLIProxyAPI upstream URL used as the default backend for the axiologic provider.
PGHOST host.containers.internal PostgreSQL host. Uses the Podman host-gateway alias by default so the container can reach the host's PostgreSQL.
PGPORT 5432 PostgreSQL port.
PGUSER postgres PostgreSQL user.
PGPASSWORD postgres PostgreSQL password.
PGDATABASE postgres PostgreSQL database name.
ENCRYPTION_KEY (empty) AES-256 hex key for encrypting API keys at rest. Auto-generated by install.sh if empty and persisted to /shared/soul-gateway/config/encryption.key.
WORKSPACE_PATH (empty) Absolute path to the Ploinky workspace. Used as a fallback to locate source code when /code is not mounted.
DEFAULT_PROXY_API_KEY (empty) API key for authenticating with the CLIProxyAPI upstream. Inherited by AXIOLOGIC_PROXY_API_KEY in startup.sh if not set separately.
DASHBOARD_PASSWORD (empty) Password protecting the web dashboard. When empty, the dashboard has no authentication.
AXIOLOGIC_PROXY_API_KEY (empty) API key for the axiologic proxy provider (CLIProxyAPI). Falls back to DEFAULT_PROXY_API_KEY.
OPENROUTER_API_KEY (empty) API key for the OpenRouter provider.
COPILOT_API_KEY no-auth-needed Dummy key for the Copilot provider. The copilot-gateway sidecar handles its own authentication via GitHub device flow.
KIRO_PROXY_API_KEY no-auth-needed Dummy key for the Kiro provider. The kiro-gateway sidecar handles its own authentication.
TAVILY_API_KEY (empty) API key for Tavily web search provider.
BRAVE_API_KEY (empty) API key for Brave web search provider.
EXA_API_KEY (empty) API key for Exa web search provider.
SERPER_API_KEY (empty) API key for Serper (Google) web search provider.
JINA_API_KEY (empty) API key for Jina web search provider (optional).

Application Defaults (config.mjs)

These values are defined in src/config.mjs. Some can be overridden by environment variables (noted below); others are compile-time constants changed by editing the source.

Provider API Key Cascade

The startup.sh script applies the following cascade logic before starting the Node.js process:

  1. If DEFAULT_PROXY_API_KEY is not set as an env var but exists in /shared/proxy_api_key, it is loaded from that file.
  2. If AXIOLOGIC_PROXY_API_KEY is not explicitly set, it inherits the value of DEFAULT_PROXY_API_KEY.
  3. If ENCRYPTION_KEY is not set but a key file exists at /shared/soul-gateway/config/encryption.key, it is loaded from that file.
Tip: For provider-specific keys (Copilot, Kiro), the gateway sidecars handle their own authentication. The dummy values (no-auth-needed) are placeholders so achillesAgentLib recognizes these providers as available.
