Middlewares

Built-in Middleware Plugins

Soul Gateway ships with 12 middleware plugins that run in the request pipeline before and after LLM dispatch. Each is a self-contained .mjs file in src/middlewares/, auto-discovered at startup.

Pre-Dispatch Middlewares

These run before the request is sent to the upstream LLM provider. They can inspect, modify, or abort the request.

pre

blacklist-scanner

Scans request message content against blacklist rules stored in the database. Blocks matching requests with a 400 status and logs the matched rule ID.

pre

rate-limiter

Enforces per-key requests-per-minute (RPM) rate limiting. Uses the key's configured rpm_limit or the setting override. Aborts with 429 when the limit is exceeded.

Default: overrideRpmLimit: null (uses key's own limit)

pre

system-prompt-injector

Injects a configurable system message into the conversation. Can prepend or append to the message array. Useful for per-tier instructions or guardrails.

Default: content: "", position: "prepend"

pre

context-compressor

Compresses long conversations by summarizing older messages when estimated token count exceeds a threshold. Keeps recent messages intact and replaces older ones with a structured summary.

Default: maxTokens: 100000, preserveRecent: 5

pre

output-compressor

Truncates verbose tool/CLI output blocks and large code blocks within messages. Keeps the first 70% and last 20% of oversized content, inserting a truncation marker.

Default: maxOutputLength: 5000

pre

cache

Returns cached responses for identical prompts (same messages + model). Should be assigned with the highest sort_order so it runs after all other pre-middlewares, hashing the final mutated messages. Non-streaming only.

Default: enabled: true

Post-Dispatch Middlewares

These run after the LLM response is received. They can inspect or modify the response content.

post

response-filter

Applies regex-based find/replace on response content for redaction or content filtering. Patterns are configured as an array of regex strings in settings.

Default: patterns: [], replacement: "[REDACTED]"

post

tpm-tracker

Tracks tokens-per-minute usage per API key by recording total tokens from the response. Non-blocking observation — errors are silently caught. Supports streaming responses.

Default: enabled: true

Both (Pre + Post) Middlewares

These run both before and after dispatch. The before() hook can set up state that the after() hook uses.

both

budget-enforcer

Pre-check: queries daily spend from call_logs and blocks requests when the budget is exceeded (429). Post-hook: updates the in-memory spend cache so subsequent requests see the latest totals. Free models are excluded.

Default: overrideDailyBudget: null

 both

loop-detector

Detects agent loops by fingerprinting responses (tool names + content prefix, SHA-256 hashed) and tracking repetition within a sliding window. Three modes: intervene (inject warning), block (429), or log (observe only). Supports streaming.

Default: mode: "intervene", similarityThreshold: 5, similarityWindow: 7
both

request-logger

Logs request and response metadata for debugging and observability. Records message count, estimated input tokens, model, tier, agent, streaming mode, response size, token usage, and middleware latency. Supports streaming.

Default: logLevel: "debug"

both

session-context

Maintains a rolling session context by extracting key facts (decisions, actions, entities) from responses and injecting them as a system message in subsequent requests. In-memory session store with configurable TTL.

Default: maxSummaryTokens: 500, sessionTTL: 3600
