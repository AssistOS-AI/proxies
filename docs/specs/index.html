Soul Gateway Specifications
Specs

Specifications

Formal requirements and design specifications for Soul Gateway. Requirements documents define what the system must do; design specs define how it does it.

Requirements

URS

User Requirements Specification

High-level user requirements describing the capabilities Soul Gateway must provide: multi-provider routing, rate limiting, observability, content safety, and extensibility.

 FS

Functional Specification

Detailed functional requirements mapping user needs to system functions: request dispatch, model resolution, streaming, cost tracking, middleware pipelines, and management APIs.

 NFS

Non-Functional Specification

Quality attributes and constraints: performance targets, security requirements, scalability limits, portability, observability, data retention, and extensibility guarantees.

Design Specifications

DS001

Request Pipeline

Full request lifecycle from HTTP ingress to response: authentication, body parsing, model routing, middleware execution, dispatch with retry, streaming, cost calculation, and logging.

 DS002

Provider Authentication

OAuth device flow and PKCE integration, adapter interface, multi-account credential pooling, token refresh loop, quota-driven rotation, and auto-provisioning.

 DS003

Middleware Framework

Plugin discovery from the filesystem, the before/after hook interface, context object anatomy, execution ordering, abort mechanics, and settings merging.

 DS004

Model Routing

Resolution algorithm with direct lookup and tier-based fallback, cooldown system for transient errors, cycle detection, per-model concurrency semaphore, and OpenRouter pricing fallback.

 DS005

Streaming

SSE pass-through from upstream providers, stream-tap for metrics capture, non-streaming buffered response, WebSocket log broadcasting, and real-time event architecture.

 DS006

Database Schema

All PostgreSQL tables, columns, types, constraints, indexes, monthly partitioning for call_logs, rate-limit state, and the middleware registry.

 DS007

Rate Limiting & Budgets

Per-key RPM/TPM sliding window implementation in PostgreSQL, budget enforcement (daily and monthly), and token-based and per-request cost calculation formulas.

 DS008

Content Filtering

Blacklist rule engine with exact, substring, and regex matching. Match actions, content concatenation strategy, and audit-safe error reporting.

 DS009

Error Handling

Error class hierarchy rooted in SoulGatewayError, retry logic with exponential backoff and jitter, error classification, cooldown triggers, and cascade rules.

 DS010

Agent Loop Detector

Behavioral loop detection via response fingerprinting and conversation growth rate analysis. Configurable intervention, blocking, and log-only modes.

 DS011

Unified Provider Auth

Direct integration of 5 provider OAuth flows (Copilot, Kiro, Codex, Gemini, Anthropic) with format converters, eliminating external gateway dependencies.
