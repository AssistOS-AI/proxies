Glossary

Key concepts and terminology used throughout the Soul Gateway system.

Soul ID

A voluntary header (X-Soul-Id) that identifies the human or system behind a request, used for per-soul log filtering and WebSocket subscriptions.

API Key

A Bearer token that authenticates client requests, stored as a SHA-256 hash with per-key rate limits, budgets, and expiry controls.

Tier

A named group of models with priority ordering that provides automatic fallback when individual models are unavailable or in cooldown.

Cooldown

A temporary blackout period applied to a model after provider errors such as rate limiting or payment failures, causing the tier resolver to skip it.

Provider

A configured LLM backend connection supporting three auth types: API key, managed OAuth with credential rotation, and internal logic.

Middleware

A processing plugin that runs before and/or after LLM dispatch, auto-discovered from .mjs files and assignable to tiers or individual models.

Format Converter

An adapter that translates between the OpenAI Chat Completions wire format and a provider-specific API protocol.

Rate Limiting

Dual-mechanism traffic control using per-key RPM (requests per minute) and TPM (tokens per minute) with a sliding-window counter in PostgreSQL.
