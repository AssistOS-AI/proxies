Provider

A configured LLM backend connection supporting three authentication types: API key, managed OAuth with credential rotation, and internal logic.

What It Is

A Provider represents a connection to an upstream LLM service. Each provider has a name, a base URL, a protocol, and an authentication mechanism. Providers are stored in the provider_configs table and linked to individual model configs via the provider_config_id foreign key.

Provider Properties

PropertyDescription
name Unique identifier for the provider (e.g., "openai", "anthropic-oauth", "copilot").
display_name Human-readable name shown in the dashboard.
protocol Wire protocol: openai, anthropic, or google. Determines how requests are formatted.
base_url Upstream API endpoint (e.g., https://api.openai.com/v1).
billing_type Authentication category: api_key, managed, or internal.
auth_type Specific auth mechanism within the billing type (e.g., "device_flow", "pkce").

Authentication Types

1. API Key (api_key)

The traditional model: the provider's API key is stored encrypted in the database and sent as an Authorization header on each upstream request.

  • Key is encrypted at rest using AES-256.
  • No token refresh or rotation logic needed.
  • Used by: OpenAI, Anthropic (direct), Google, OpenRouter, Groq, Fireworks, DeepSeek, etc.

2. Managed OAuth (managed)

OAuth-based authentication with support for multiple accounts and automatic credential rotation. Managed by the auth-manager.mjs system.

  • Device flow: User-interactive authorization (e.g., GitHub Copilot). Returns a userCode and verificationUri for the user to visit.
  • PKCE flow: Browser-redirect authorization (e.g., Anthropic OAuth). Returns an authUrl for the user to open.
  • Multi-account: Multiple OAuth accounts can be registered per provider, enabling credential rotation.
  • Token refresh: A background loop (every 60 seconds) checks for expiring tokens and refreshes them proactively, with a configurable margin (default 60 seconds before expiry).
  • Quota rotation: When an account's quota is exhausted, rotateAccount() marks it as exhausted and switches to the next available account. Exhausted accounts auto-reset at midnight UTC.
  • Used by: Copilot Gateway, Kiro Gateway, Anthropic OAuth, OpenAI Codex OAuth.
// From auth-manager.mjs — credential retrieval with auto-refresh
export async function getCredentials(providerName) {
  const adapter = adapters.get(providerName);
  if (!adapter) return null;

  const accounts = await store.readAccounts(providerName);
  if (accounts.length === 0) return null;

  // Find active, non-exhausted account
  let account = accounts.find(a => a._index === state.activeIndex);
  if (!account || account.quotaExhausted) {
    account = accounts.find(a => !a.quotaExhausted);
    if (!account) return null; // All exhausted
  }

  // Refresh if expiring soon
  if (account.expiresAt && Date.now() + refreshMarginMs > account.expiresAt) {
    await refreshAccountToken(providerName, account._index, adapter, account);
  }

  return {
    token: account.accessToken,
    headers: adapter.getHeaders ? await adapter.getHeaders(account) : {},
    formatConverter: adapter.formatConverter || null,
  };
}

3. Internal (internal)

Built-in logic that does not call an external API. The provider handles the request entirely within the gateway process.

  • No credentials needed.
  • Used by: search aggregator (web search across multiple providers).

Auth Adapters

Each managed provider registers an adapter with the auth manager. An adapter implements:

Method / PropertyDescription
name Unique adapter name matching the provider name.
authType "device_flow" or "pkce".
startAuth() Initiates the OAuth flow. Returns device code info or a PKCE auth URL.
pollForToken(deviceCode, interval) Polls the token endpoint for device flow completion.
exchangeCode(code, state) Exchanges an authorization code for tokens (PKCE flow).
refreshToken(account) Refreshes an expired access token using the stored refresh token.
getHeaders(account) Builds the HTTP headers for upstream requests (e.g., Bearer token, custom headers).
formatConverter Optional format converter name for providers that don't use the OpenAI wire format.
providerTemplate Template for auto-provisioning the provider_configs DB row.
knownModels Array of model IDs to auto-provision in model_configs after first login.

Auto-Provisioning

When a managed provider is authenticated for the first time (or on startup via reconcileProviders()), the gateway automatically:

  1. Creates a provider_configs row from the adapter's providerTemplate (display name, protocol, base URL, billing type).
  2. Creates model_configs rows for each entry in the adapter's knownModels array, using the axl/<provider>/<model> naming convention via buildModelName().

This is idempotent — the upsert logic updates existing rows rather than creating duplicates, and it runs safely after every OAuth login.

Account Status

The getAuthStatus() function returns the health of a managed provider's accounts:

StatusMeaning
activeAt least one account is authenticated and has quota.
expiringThe active token is about to expire (within refresh margin).
all_exhaustedEvery account has hit its quota limit. Resets at midnight UTC.
needs_reauthToken refresh failed; the account needs a new OAuth flow.
no_accountsNo OAuth accounts have been registered for this provider.
